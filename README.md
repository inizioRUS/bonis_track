# Agent-Enhanced Corporate Search (RAG + Agents)

## Описание проекта

Проект направлен на улучшение корпоративного поиска по внутренним артефактам компании (Confluence, Jira, кастомные документы и др.) с использованием RAG-архитектуры.

Целевая аудитория:
- разработчики
- аналитики
- продакт-менеджеры
- внутренние команды

### Текущая боль

- Данные распределены по множеству источников
- Сложные и длинные запросы имеют низкий recall
- LLM может генерировать hallucinations
- Пользователи делают повторные уточняющие запросы
- Снижается доверие к системе

Цель — повысить полноту и надежность ответов при сохранении приемлемой latency и стоимости.

---

## Что делает PoC

- 🔎 Поиск по нескольким источникам (wiki + mock Jira)
- 🧠 Базовый RAG (baseline)
- 🤖 Агентная логика:
  - декомпозиция сложного запроса
  - multi-step retrieval
  - верификация ответа
- 📊 Сравнение baseline vs agent по:
  - Recall@k
  - MAP
  - nDCG
  - качество ответа (LLM-as-judge)
  - latency
