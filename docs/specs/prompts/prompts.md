# Prompts

Документ фиксирует базовые промпты для ключевых агентов PoC-системы.  
Промпты нужны как часть архитектурной спецификации: они показывают, как именно LLM включена в workflow, какие ограничения на неё наложены и какой structured output ожидается на каждом шаге.

---

## Общие принципы

Для всех агентных промптов действуют общие правила:

- агент работает только в рамках своей роли;
- агент не должен выходить за пределы своего контракта;
- агент не должен придумывать tool names или аргументы;
- агент не должен выполнять side effects напрямую;
- все критичные шаги должны завершаться structured output;
- orchestrator валидирует ответ агента перед следующим переходом.
